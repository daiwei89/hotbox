How Hotbox works

In practice, Machine Learning (ML) needs extensive feature engineering to work well. Often it's achieved by combining various "views" of the original features, and thus originally small set of features (e.g., thousands) could become very high dimensional (e.g., billions):

Hotbox supports transforms from one-hot encoding, cartesian products to gradient boosted tree and neural network transforms (by working with external training systems):

Why Hotbox?

  • Ultra High Dimensional: Hotbox can generate up to billions of features efficiently.
  • Petuum Integration: Hotbox works with Petuum systems such as Bosen and Strads. Feature engineering and training in one performant eco-system.
  • Speed: Hotbox transform data on the fly (a.k.a streaming) instead of waiting for all the data to be loaded. No problem if data doesn't fit in memory. Also transformed features are often larger than the raw features. By not storing it we save disk at small compute cost. In contrast, Spark feature transforms only works in batch mode. Dataset exceeding memory size will likely to be very slow.
  • Lightweight: Hotbox encodes data smartly and compress before storing, reducing dataset size by 2~5x. For ML applications whose runtime dominated by disk IO time, this is a huge saving. In our experiment Hotbox-equipped system can load data 36% faster than Spark loading from Hive.
  • Convenience: Simple C++ and Python API to integrate with your ML application. Fully support distributed training and worker failure and elastic scaling.
  • ML support: Come with a library of common transforms for ML applications that configurable for your app.
